3. Always add a sensible LIMIT (default 100) unless the question is an aggregate.
4. Use ISO date literals and Snowflake date functions (DATEADD, DATE_TRUNC).
5. If the question cannot be answered from these objects, output exactly:

6. Output ONLY the SQL (or the CANNOT_ANSWER comment). No markdown, no prose.
